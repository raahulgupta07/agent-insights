System Credentials (Service Principal)

📘 How to get each value

Microsoft Fabric uses a Service Principal (an Entra ID app registration) for headless access. Set it up once, then fill the 6 fields below. Hover a field to highlight its guide.
One-time setup (do first): in Microsoft Entra ID → App registrations → New registration create an app — it gives you the Tenant ID, Client ID & Client Secret. Then in Fabric, grant that app access (see Client Secret card).
Server Hostname REQUIRED
The SQL analytics endpoint host of your Warehouse / Lakehouse.
Fabric portal → open your Warehouse (or Lakehouse) → ⚙ Settings → SQL analytics endpoint → copy the SQL connection string host.
abc123xyz.datawarehouse.fabric.microsoft.com
Use only the host — drop any ,1433 port or database suffix.
Database REQUIRED
The Warehouse or Lakehouse name, exactly as it appears in your Fabric workspace.
Fabric workspace → the item name shown next to the 🗄/🏠 icon.
SalesWarehouse
Schema OPTIONAL
Limit discovery to one or more schemas. Leave empty to discover all.
Default is dbo. For several: dbo, sales, finance
Tenant ID REQUIRED
Your Azure / Entra directory (tenant) GUID.
Azure portal → Microsoft Entra ID → Overview → copy Tenant ID. (Also on your app's Overview as “Directory (tenant) ID”.)
72f988bf-86f1-41af-91ab-2d7cd011db47
Client ID REQUIRED
The app registration's Application (client) ID.
Entra ID → App registrations → your app → Overview → Application (client) ID.
1a2b3c4d-...-9z8y7x
Client Secret REQUIRED
A secret VALUE created on the app registration.
  1. Entra ID → App registrations → your app → Certificates & secrets
  2. New client secret → set an expiry → Add
  3. Copy the Value column immediately (not the Secret ID)
The Value is shown once — if you miss it, create a new secret.
Then grant access in Fabric: Workspace → Manage access → add the app as Member/Contributor. Admin must also enable “Service principals can use Fabric APIs” in the Fabric/Power BI tenant settings.
Stuck on Test Connection? 90% of failures = the service principal wasn't granted access to the workspace/warehouse, or the tenant setting above is off.
