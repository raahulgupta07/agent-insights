2 · How they wire into the agent (and at which scope)

One question flows through the agent loop. Each feature plugs into a specific stage. Scope = who a given fact reaches.

① Question comes in (composer, ⑦ optionally via a Workflow replay)

Part A · Workflow replay→ substitutes params→ re-runs saved steps headless
Scope: a workflow is AGENT/org (scope=org) or private.

② Pick the right tables

semantic relevance+ P2 usage-trust blend→ ranked tables
Trust = verified queries + dashboards + goldens for THIS data source. Scope AGENT.

③ Build context for each table (the heart)

P3 Table Card = schema+values· C: PK / used-by / alt-table· freshness/owner/PII· + Shared-Memory overlay
Overlay pulls corrections at 3 scopes: GLOBAL (all agents) · BY-DATA (same model/schema) · PRIVATE (you). B pre-builds this card nightly.

④ Ground with company knowledge

P4 Institutional-KB→ approved docs (incl. E: Notion/Slack)→ metric defs into planner
Scope GLOBAL org + approved only (no per-viewer ACL yet).

⑤ Model answers → SQL runs (read-only) → result

planner + tools→ create_data (read-only guard)→ answer + dashboard

⑥ Learn + guard (closes the loop)

👍 → Shared Memory capture· D: golden-SQL + rows saved· P5: canary health/drift
Captured learning re-enters ③ next time. Scope of a new learning is decided by the sanitizer (data stripped) + resolver.

3 · Global vs per-agent vs per-user — who sees what

🌐 GLOBAL org

Every agent in the org reads it.

Shared-Memory global tier (e.g. “exclude total rows”)
P4 institutional docs (approved)
P5 eval health, D golden-SQL results
🧩 PER-AGENT data source

Only agents on the same data (same PBI model / DB schema / file).

P2 usage-trust scores
P3 table cards + C enrichment
By-data Shared-Memory tier
Workflows with scope=org
🔒 PER-USER private

Never crosses to another user.

Private Shared-Memory tier
Private workflows (scope=private)
Per-user connector clones (existing)

5 · The two UI gaps worth building

1
Report header “Save as workflow” button — completes Part A (backend live). Turns any finished analysis into a reusable workflow.
2
Settings page “Notion / Slack sources” — token + “Sync now” + last-synced status → calls the existing /api/kb-sources/* endpoints. Without it, Part E can only run from an API call.
